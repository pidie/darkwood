Test names should follow the following naming convention:

- The name of the test will have four sections
- All words in a section will use Pascal casing
- All sections will be separated by an underscore
    - Section one will be the name of the script being tested
    - Section two will be the name of the method being tested in said script
    - Section three will be a brief description of the test
    - Section four will be the expected result of the test

----------
example:

NameOfScriptBeingTested_NameOfMethodBeingTested_DescriptionOfTest_ExpectedResult
